## 1. Общее описание

Реализован инструмент для построения **транзитивного графа зависимостей** пакетов с использованием **итеративного DFS (без рекурсии)**.  
Поддерживается:
- Анализ с **ограничением глубины**.
- **Обнаружение и обработка циклических зависимостей**.
- **Тестовый режим**: вместо URL репозитория — локальный файл с графом, где пакеты — **большие латинские буквы** (A, B, C и т.д.).

> **Цель**: построить граф, обработать транзитивность, циклы и глубину.

---

## 2. Описание функций и настроек

| Функция | Описание |
|--------|--------|
| `get_transitive_dependencies()` | Итеративный DFS через стек. Возвращает множество зависимостей и флаг цикла. |
| `load_graph_from_file()` | Загружает граф из `.txt` файла формата `A: B C`. |
| `--package` | Имя стартового пакета (обязательно). |
| `--repository` | Путь к `.txt` файлу (в тестовом режиме). |
| `--repo-mode` | `local` (по умолчанию) — работа с файлом. |
| `--depth` | Максимальная глубина анализа (по умолчанию 1). |

> **Циклы**: обнаруживаются по наличию узла в текущем пути DFS. Циклические зависимости **не добавляются** в результат, но флаг `cycle_detected = True`.

---

## 3. Команды для запуска и тестирования

```bash
# Запуск Этапа 3 (тестовый режим)
python script3.py \
  --package A \
  --repository test_graph.txt \
  --repo-mode local \
  --depth 3
